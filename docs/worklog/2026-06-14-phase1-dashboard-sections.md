## 設計上のポイント

- **ロジックは一切変更していない**。JavaScript は `getElementById`（`$('sc-xxx')`）でDOMを参照するため、
  要素の ID と属性さえ保てば、DOM の配置（どのカードに属するか）を変えても動作は不変。
  → このフェーズは純粋な「DOM の引っ越し + カード見出しの付与」。
- 接続バッジ（`sc-conn`）は元々カード内にあったが、「どのカテゴリにも属さない全体状態」なので
  タブ最上部の `.sc-tabhead` に移動。
- `sc-err`（エラー表示）はカード外・タブ末尾に移動。`scControlErr()` は ID 参照なので位置自由。
- ⑤デバイス調整は**まだカードを出さない**（中身が無い空カードは不格好）。コメントで位置だけ確保し、
  フェーズ2で `<section class="card">` ごと追加する。

## 用語メモ

- **status_api.py (:8080)**: ダッシュボードのバックエンド。`/dashboard` で HTML を返し、
  `/status`・`/control/*` をブラウザと gateway(:8767) の間で中継する薄いプロキシ。Python標準ライブラリのみ。
- **`.ctl` / `.card`**: ダッシュボードの基本UI部品クラス。`.card` が枠（角丸ボックス）、
  `.ctl` がその中の1コントロール行。今回はこれらを再利用しただけで新規スタイルは最小（見出し用2クラスのみ）。
- **「骨組み先」方針**: 機能を足すたびにレイアウトを作り直すと二度手間になるため、
  先にカテゴリの器を作る → 各フェーズで中身を入れる → 最後にまとめてUXを仕上げる、という順序。

## 追記: デザイン刷新（ユーザーフィードバック対応）

骨組み再編の直後、ユーザーから「崩れはないがデザイン・配置がつまらない。音量バーの横にミュートを
小さくまとめられそう」とのフィードバック。骨組み（フェーズ1）の段階で**コントロールの見た目ルール
（デザインの型）を確立**し、後続フェーズの新項目（明るさ/LED等）も同じ型に載せる方針で実装した
（→ 最後のフェーズ5は微調整だけで済む）。方向性は「コンパクト＆洗練＋スタックちゃんぽさ」を選択。

### 確立した「デザインの型」（CSS コンポーネント、再利用前提）

- `.row` … ラベル＋現在値を両端に置く行（`justify-content:space-between`）
- `.row-label` … アイコン＋テキストのラベル
- `.row-val` … 右寄せの**大きい現在値**（18px/bold/tabular-nums）。値を強調して単調さを解消
- `.slider` … スライダー＋末尾の小コントロール（ミュート等）を横並びにする行
- `.icon-btn` … 小さいアイコンボタン。`aria-pressed="true"` で赤くなる状態表示
- `.btn-primary` … 主要アクション（聞き取り開始）を緑寄りで強調
- `.sc-ava` … **StackChan の顔アバター**（四角い画面＋2つの目）。`.on` で目が緑に光る

### 主な変更

- 🔊 **音量＋ミュートを1行に統合**: スライダー右端に `.icon-btn` のミュート。ミュート中は🔇＋赤。
  → ミュートを checkbox トグルから**ボタン化**したため JS も変更（後述）。
- 各コントロールを `ラベル＋大きい現在値 → スライダー` の統一レイアウトに（マイク/感度/近接/表情/heartbeat/通知）。
- 🤖 タブ上部に**顔アバター**を新設し、接続中は目が緑に光る（接続状態の即時把握）。

### JS の変更点（HTML の型変更に伴う最小限）

- ミュート: `change` リスナー廃止 → `scToggleMute()`（click）＋ `setMuteUI(muted)`（aria-pressed と🔊/🔇切替）。
  `loadStackchan` の同期も `$('sc-mute').checked=...` → `setMuteUI(...)` に変更。
- 接続表示: `loadStackchan` 成功/失敗の両方で `$('sc-ava').classList.toggle('on', connected)` を追加。
