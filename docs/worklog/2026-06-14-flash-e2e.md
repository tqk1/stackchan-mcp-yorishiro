### 2. 前回失敗の真因と今回の解決

| | 前回（持ち越しの原因） | 今回 |
|---|---|---|
| 症状 | `OSError: [Errno 71] Protocol error`（EPROTO）。esptool の自動リセット(RTS/DTR)を develop 版ファームがブロック | まず `/dev/ttyACM0` 不在・`lsusb` に Espressif 無し（デバイス未認識） |
| 対処 | `--before no-reset` 等を試すも「No serial data received」で延期。sdkconfig 恒久対策を検討予定だった | **USB-C を物理抜き差し**して再接続 → `303a:1001` 出現 |
| 結果 | 未完了 | `--before default_reset` がそのまま通り **成功** |

**学び**: EPROTO も「ポート不在」も、まず **USB 抜き差し**で切り分ける。
懸念していた `CONFIG_ESP_CONSOLE_USB_SERIAL_JTAG` の sdkconfig 恒久対策は**不要だった**。
ModemManager は active のままで flash 成功 → EPROTO の原因ではないと再確認。

### 3. flash 実行（app + assets、NVS 保持）

今回は `config.json`（ウェイクワード単一ワード復帰）が assets に乗るため、app だけでなく assets も焼いた。

```bash
cd firmware
~/.venvs/esptool/bin/esptool.py --chip esp32s3 --port /dev/ttyACM0 -b 460800 \
  --before default_reset --after hard_reset \
  write_flash 0x20000 build/xiaozhi.bin 0x800000 build/generated_assets.bin
```

- `xiaozhi.bin` @0x20000（app）→ **Hash verified**
- `generated_assets.bin` @0x800000（assets）→ **Hash verified**
- `0x0`（ブートローダ/パーティション/**NVS**）は不触 → 首中立姿勢・近接設定・閾値は保持

## 構成図

### flash の経路（ビルドは Docker、書き込みはホスト）

```
firmware/ (ソース)
   │  docker run espressif/idf:v5.5.2  python ./scripts/release.py stackchan
   ▼
build/xiaozhi.bin (app) / build/generated_assets.bin (assets)
   │  ~/.venvs/esptool (ホスト, /dev/ttyACM0)   ※Docker からは焼かない
   ▼
CoreS3 flash:  0x20000=app   0x800000=assets   0x0=NVS等(不触)
```

### E2E のデータフロー

```
[画面タップ] → firmware 録音 → WS → gateway
   → faster-whisper(STT) → ルーティング(短文=ローカル LFM2.5 / それ以外=Hermes)
   → VOICEVOX(TTS) → WS → firmware 再生
   ※この一連の間 voice_turn_active=True → heartbeat は tick 全体をスキップ
```

## 用語

- **EPROTO (Errno 71)**: esptool がシリアルの RTS/DTR を切り替えて ESP を自動リセットしようとした時、
  ファーム側の USB-Serial/JTAG 設定が噛み合わず ioctl が拒否されるエラー。USB 抜き差しで回復することがある。
- **NVS (Non-Volatile Storage)**: ESP32 のフラッシュ内の不揮発キーバリュー領域。首中立姿勢・近接閾値などを保存。
  `0x0` を焼くと消えるが、`0x20000`/`0x800000` だけなら保持される。
- **MultiNet**: assets に同梱される中国語ピンインのウェイクワード認識モデル。日本語「スタックちゃん」の
  認識限界が判明済み（②はクローズ、タップ/背面なで運用）。
- **voice_turn_active**: gateway の会話進行中フラグ。STT→Hermes→TTS の全区間で True。
  heartbeat はこれを見て会話中の割り込みを避ける（設計原則①）。
